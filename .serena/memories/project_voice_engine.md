## Модель
- `VoiceTranscript` (`voice_transcripts`): джоб и результат одной строкой — `fileId @unique` (FK на FileObject, onDelete Cascade) → **1 файл = 1 транскрипт навсегда** (Telegram-модель «Расшифровать», кэш вечен; language влияет только на первый расчёт/после error). Поля: status queued|processing|ready|error, language/detectedLanguage, text, segments Json [{start,end,text,speaker?}], durationMs Int (не BigInt), diarize, attempts, requestedById, **leaseUntil** (аренда джоба). Индексы: [status,updatedAt] (queued-редрайв) + [status,leaseUntil] (processing-редрайв).
- `VoiceRecording` (`voice_recordings`): сущность Диктофона (ownerId, title, source upload|web|terminal, language; **durationMs-колонки НЕТ** — длительность = meta файла с добором из транскрипта, один источник правды); файл через FileLink refType=`voice_recording` (несвязанный ready-файл реапится — поэтому запись создаётся СРАЗУ после upload).

## Движок (apps/api/src/core/voice/, @Global)
- **Джобы**: клейм status-guarded `updateMany {fileId,status:'queued'}→processing` (count!==1 → выход) → **аренда**: `leaseUntil = now + sttJobOverheadMs(25мин) + sttTimeoutMs(этого джоба) + sttLeaseMarginMs(2мин)` — живой джоб не переживёт свой бюджет (у скачивания/prep/STT свои таймауты). **Все финальные записи (ready/requeue/error) гардятся клейм-токеном `attempts`** — поздняя запись отставшего джоба не заденет переклеймленную строку. Redis-лок `voice:stt:<fileId>` TTL 60 мин (> худшего бюджета). Ошибки: transient → назад в queued (≤3 попыток), terminal → error + событие. `VoiceCron` */2 мин: потерянные queued (>2 мин) и processing с протухшей арендой (leaseUntil<now; legacy без аренды — по worst-case бюджету от updatedAt).
- **Infected-гейт**: `requestTranscript` бросает 400 на scanStatus=infected (не ставит обречённый джоб); `execute` на infected → terminal error сразу (не гоняет 3 попытки об ForbiddenException).
- **HTTP-таймаут STT** = min(30 мин, 120с + 3×durationMs); fallback от размера ~1 мин/МБ; **считается ОДИН раз при клейме** (от meta.durationMs/size) — аренда всегда ≥ таймаута.
- **Байты**: local-драйвер → `files.localPathFor(storageKey)` прямой путь (без стрим-копии 200МБ), s3 → openRawStream в tmp. Строка+файл — одним запросом (`findUniqueOrThrow include file`).
- **Драйверы** (`voice-stt.client.ts`, паттерн process-ai-client): `openai_compatible` — multipart POST `{VOICE_STT_URL}/v1/audio/transcriptions` (`/v1` дописывается, если не задан), `response_format=verbose_json`, Bearer `VOICE_STT_API_KEY`, `fs.openAsBlob` (стрим с диска) с фолбэком на readFile; поле `speaker` мапится защитно; **таймаут — `AbortSignal.timeout`** (ловим TimeoutError И AbortError). `mock` (`VOICE_STT_MOCK=true`) — канон-текст на 2 спикеров, без сети/ffmpeg (CI). **mock ПЕРЕКРЫВАЕТ URL**. Модель под язык: kk + `VOICE_STT_MODEL_KK` → казахская. Мёртвые капабилити-геттеры (diarization/languages) удалены — `/voice/status` = `{enabled, mock}`.
- **Подготовка** (`voice-audio.ts`): ffmpeg-static → 16 кГц mono wav + `highpass=f=80,afftdn=nf=-25,dynaudnorm` — только для STT, играбельный оригинал НЕ трогаем; нет бинарников → оригинал в STT (whisper сам декодит). `prepareForStt(…, knownDurationMs)` — длительность из meta файла, ffprobe только когда её нет. **Обвязка ffmpeg — общая `shared/ffmpeg/ffmpeg.util.ts`** (ffBinaries/execFF(timeoutMs)/ffprobeFormat) — одна копия с конвейером files, таймауты параметром (prep 10мин, конвейер 60с). Карта mime→ext — shared `AUDIO_MIME_TO_EXT` (вкл. audio/mp3, audio/x-flac).
- **Доступ** = доступ к файлу: `files.getMeta(viewerId, fileId)` (403/404 через резолвер привязки).
- **Сервисные API потребителям** (в таблицы движка снаружи не ходим): `getStatusesForFiles(fileIds)` → Map{status,durationMs} (батч для списков) · `deleteForReapedFiles(fileIds)` — deleteMany с `file.status='deleted'` в WHERE: транскрипт умирает ТОЛЬКО с файлом, шаренный (голосовое в чате ← запись Диктофона) сохраняет общий транскрипт.
- **API**: GET `/voice/status` {enabled,mock} · POST `/voice/transcripts` {fileId,language?,diarize?} (идемпотентно, P2002-catch) · GET `/voice/transcripts/:fileId` · POST `/voice/stt` (multipart ≤25МБ, sync — фундамент AI-команд/SuperTerminal6). События `voice.transcript.ready|failed` несут **`links` [{refType,refId}]** (через `files.listLinksOfFile`) — потребители фильтруют свои события по payload, без запроса в fileLink на каждый чужой транскрипт.

## Волна (конвейер files)
`processAudio(spec, source)`: по **капабилити профиля `waveform: true`** в FILE_PROFILES (voice_message, dictaphone — НЕ хардкод имён в движке; 8-й аудио-потребитель = флаг в профиле). PCM-декод `-ac 1 -ar 2000 -f s16le` **во временный файл** (`execFF` возвращает stdout строкой — бинарь через него нельзя!) → 96 RMS-бакетов → пик-нормировка 0..100 → `meta.waveform`. **Кап ≤10 мин fail-closed**: известная длительность > капа → скип; НЕизвестная (webm MediaRecorder без заголовка — браузерные записи!) → декод ограничен `-t капом`, реальная длительность добирается из PCM-байтов в `meta.durationMs` (закрывает duration=Infinity на вебе); упёрся в кап (truncated) → ни волны, ни длительности. Ошибка волны не валит конвейер.

## Потребители
- **Мессенджер**: кнопка 🎤 (`VoiceRecordButton`) → `useVoiceRecorder` (MediaRecorder, constraints echoCancellation/noiseSuppression/autoGainControl, каскад mime webm/opus→webm→mp4 (Safari)→ogg; **`stop()` на inactive-рекордере (гарнитура отвалилась/трек кончился) собирает уже накопленные timeslice-чанки** — запись не теряется и промис всегда резолвится) → upload профилем `voice_message` → **существующий** attachment-путь `onSendAttachmentsRef` (нового эндпоинта НЕТ). Классификация «голосовое» — shared `isVoiceNoteProfile`; превью — shared `attachmentPreviewText` (одна точка API+веб). `VoiceMessageBubble`: волна клик-seek, скорость ×1/1.5/2; **duration=Infinity (headerless webm без meta.durationMs) лечится сик-трюком** (currentTime=1e7 → durationchange с конечным значением → сброс на 0).
- **Поллинг транскрипта — общий хук `useVoiceTranscript(fileId, {enabled})`** (`apps/web/src/lib/hooks/`): запрос ask() (идемпотентный POST → setQueryData) + **СВОЙ setTimeout-цикл по `dataUpdatedAt`** с бэкоффом 2с (<30с) → 5с (<3мин) → 15с (VOICE_LIMITS.pollInterval*). Бабл: `enabled: requested` (клик) — без гарантированных 404 на каждый бабл; Диктофон: `enabled: transcriptStatus != null || asked`. Один механизм на все поверхности.
- **Диктофон** `modules/recorder` + веб `/recorder` (одностраничник — вне ServiceShell осознанно, переедет вместе с разделами протоколов): refType `voice_recording` (owner-only, allowedProfiles dictaphone+voice_message), профиль `dictaphone` 200MB (== hardMaxSize — потолок НЕ поднимали: validation/file.ts + multer + ClamAV compose). Бэкенд: list через `voice.getStatusesForFiles`, rename → лёгкий `{id,title}` (веб патчит кэш), remove → `files.getLinkedFileIds` + `unlinkAllForRef` + `voice.deleteForReapedFiles`. Веб: загрузка через `useFileUpload`×2 (source upload|web) + `UploadProgressList`; дип-линк `?id=` под гардом **`isPending`** (НЕ isLoading — у disabled-запроса isLoading=false и эффект затирал линк); деталь под `key={rec.id}` (язык/requestError не утекают между записями); авто-стоп записи с ре-энтри-гардом (finishingRef); `TranscriptView` — метки через общий `formatDuration` (умеет h:mm:ss). Уведомления `voice.transcript.ready|failed` (фильтр по payload.links, actionUrl `/recorder?id=`).

## Грабли (пойманы при стройке и ревью)
- **MIME-вариации браузеров (поймано юзером на реальном mp3)**: Windows-Chromium отдаёт `.mp3` как `audio/mp3`, не `audio/mpeg` → 400 на init профиля. Фикс: `audio/mp3`+`audio/x-flac` в AUDIO_MIME; веб-нормализация пустого/octet-stream MIME по расширению — `normalizeAudioMime` (карта — shared `AUDIO_EXT_TO_MIME`); `.webm` ОС регистрирует как ВИДЕО (video/webm) → accept с явными расширениями + ремап video/webm→audio/webm. «Расшифровать» — у ЛЮБОГО аудио-вложения чата (`TranscriptBlock` в AudioTile). Формат записи: Opus 48 кГц @ 64 кбпс; STT-препроцессинг → 16 кГц mono wav.
- **React Query поллинг (ревью 2026-07-16, проверено в браузере)**: `refetchInterval`-функция НЕ перевзводится после `setQueryData` — И invalidateQueries НЕ спасает надёжно (поллинг вставал после первого refetch). Решение: поллинг СВОИМ setTimeout-циклом по `dataUpdatedAt` в `useVoiceTranscript` — детерминирован, RQ-семантика не при делах. Старая заметка «invalidate обязателен» — недостаточна.
- **RQ v5 isLoading ловушка**: у disabled-запроса (enabled: isReady при гидрации auth) `isLoading=false` при пустых данных → гард «пока грузится» должен быть на **`isPending`**.
- PowerShell: `$env:X='y' && cmd` — parse error, только `;`; `$disconnect` в `node -e` съедается PS.
- Browser pane блокирует микрофон → состояние `denied` проверено, реальная запись — ручная QA (Chrome/Firefox/Safari).
- verify: STT-проверки только на mock (live непредсказуем, фикстура — синус); wav-фикстура синтезируется в скрипте (RIFF-заголовок + синус), magic-bytes sniff пропускает.
