## Round-2 findings — ComfyUI lessons (concepts only; core GPL-3, litegraph MIT)
- **Schema-driven canvas**: server serves all node defs via GET /object_info; UI knows only what defs declare. Node def = INPUT_TYPES classmethod (required/optional/hidden; hidden = engine-injected context like node id/whole graph), RETURN_TYPES/NAMES, FUNCTION, CATEGORY, OUTPUT_NODE.
- **Typed sockets** = plain strings, equality match, color-coded; UI refuses incompatible connections; drag-to-empty opens type-filtered node search; server re-validates on submit. **Widget↔socket duality** (the killer UX): every primitive param renders as in-node widget (default/min/max/step, multiline, combo) OR via forceInput becomes a connectable port — user-switchable.
- **Execution**: async queue; pull-based from output nodes (dead branches never run); **per-node output caching by input-signature of the whole upstream cone** (+ IS_CHANGED hook; change one widget → only that node + descendants re-run) — for LLM canvases this is token-cost control in editor test-runs; lazy inputs skip un-needed branches; mute(2)/bypass(4) node modes; per-node WS events (executing/progress/executed/execution_error → red node with message in place).
- **Two JSON formats**: UI workflow (positions/links/groups/notes/widgets_values; widgets_values positional = known fragility → we use named) vs compiled **API prompt IR** `{nodeId:{class_type, inputs:{param: literal | [srcNodeId, outputIdx]}}}`; frontend compiles UI→IR (folds widgets, prunes muted, rewires bypassed, flattens subgraphs/reroutes). PNG metadata embeds the workflow = every artifact carries its recipe.
- **Subgraphs** (2025): collapse selection into super-node with exposed ports/promoted widgets, nestable — the reuse/template unit. **Nodes 2.0**: ComfyUI migrated node rendering from canvas-drawn litegraph to Vue DOM components (canvas rendering was a bottleneck) → validates @xyflow/react (DOM from day one).
- Registry lessons: globally-unique node type names, immutable semver, snapshot/lockfile, «install missing nodes»; B2B inversion: curated first-party catalog only, «custom node» = declarative manifest, NEVER tenant code in-process.

## Architecture (locked)
- `modules/processes` + @Global node registry (5th platform registry). **Node descriptor**: stable type id, version, RU title/description (agent-readable), CATEGORY, typed ports in/out (flow + resource families: ai_model/ai_memory/ai_tool/knowledge...), config schema (Zod→JSON Schema; each param widget-or-port ComfyUI-style), capability required, dangerous flag, examples. One registration → canvas UI + validation + tool-exposure + (позже) MCP.
- **Two formats**: ProcessVersion stores canvas document (nodes/edges/notes/positions, named widget values); publish = compile→validate→execution IR snapshot (literal | [nodeId, port] refs). Instance pins compiled IR.
- **Engine**: DB-row token engine (ProcessDefinition→ProcessVersion→ProcessInstance(анкета JSONB, wakeAt, status)→ProcessStepRun(input/output snapshots, claim/timestamps, taskId?)+ProcessTrigger(+cursor state for polling)). Idempotent executor, status-guarded updateMany, wakeAt+Redis-lock cron sweep. Per-node live status via socket.io (ComfyUI executing/progress/error events). Data refs between nodes = pill-picker paths `{{nodes.find_machine.price}}` (no JS eval). Human forms = анкета instance form (Kissflow), per-step field perms.
- **Agent node** = root with sub-node ports: Модель (shared across agents — CheckpointLoader pattern), Память (Postgres chat memory, session key), Инструменты (= ANY registered node via $fromAI-style LLM-filled params + HTTP tool + RAG tool + Agent-as-tool for hierarchies), structured-output parser; maxIterations, dangerous-tool HITL gate (engine-routed tool loop: agent yields toolCalls → engine runs tool nodes as steps → resume).
- **RAG**: «База знаний» on pgvector (our PG16): ingestion (upload→splitter→embeddings→vectors, collections per workspace), nodes: knowledge.search (step) + knowledge-as-agent-tool. Embeddings via API (Voyage/OpenAI).
- **Credentials vault**: per-workspace encrypted (AES-GCM) typed credentials, referenced by id from nodes, never inline in document.
- Human steps: human.task (real Task via TasksService, ModuleRef sync hook like ShopService), human.approval (rich-card buttons; + Telegram/WhatsApp buttons later), dept queue+claim («Входящие отдела»), SLA+escalation, per-dept timing report (GROUP BY ProcessStepRun).
- Access: resource type `process` (editor/starter/viewer), edit=manager+, dev-only processes = admin+-restricted visibility. Money nodes later (sync escrow only).
- Canvas: @xyflow/react; palette by category; NDV-style config panel; type-colored ports; drag-to-empty type-filtered search; sticky notes; mute/bypass later; subgraphs/templates later.
