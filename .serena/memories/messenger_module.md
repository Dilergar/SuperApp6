# Messenger Module (Мессенджер)

Communication bus + embeddable contextual chats + interactive Rich Cards + presence + mentions. Built phase-by-phase; GRILL before each phase. Full design + rationale in Claude auto-memory `project_messenger_design.md`.

## Architecture
- WEB-FIRST. Mobile + offline-first = late phases.
- Realtime = **socket.io** (WS + fallback + reconnect + rooms + @socket.io/redis-adapter), ns `/messenger`, JWT on handshake via process.env.JWT_SECRET (NOT @nestjs/config). RedisIoAdapter in main.ts.
- ONE unified `Chat` (type dm|group|context). Access via core/access `chat` type: member + viewer=union(this,computed(member)). DM/group = direct tuples chat#member@user; context (task/order/event) = usersets chat#member@<type>#<role> (entity roles = source of truth → unassign = instant Hard Revoke). Caps chat.view/chat.post.
- Read receipts = cursors in ChatMember (deliveredSeq + lastReadSeq); ticks DM-ONLY. Server-readable (NOT E2E). System messages type='system' authorId=null, excluded from unread.
- Per-chat monotonic seq via Chat.lastSeq in a $transaction.
- **CRITICAL pattern: roles project SYNCHRONOUSLY at the domain mutation** (TasksService.resyncTaskRoles, ShopService.resyncOrderRoles in buy/contribute/withdraw, messenger getOrderChat/getEventChat call resync before access check). EventBus listeners are idempotent SAFETY NETS. resync*Roles DIFF-BASED (applyDiff), NOT revoke-then-grant (no transient-empty 403 window).

## Schema
- Chat: type, dmKey(unique DM), parentType/parentId(@@unique, one chat per task/order/event), createdById(group owner), workspaceId, lastSeq, title.
- ChatMember: role(owner|admin|member|bot), visibleFromSeq(0=full history), deliveredSeq/lastReadSeq(+At), mutedUntil, pinned, archived, leftAt, @@unique[chatId,userId].
- Message: type(text|system|rich_card), content?, payload?(rich_card=RichCardPayload), authorId?(null=system), seq, editedAt, deletedAt, @@unique[chatId,seq].
- **Mention (Ф5): mentionedUserId, mentionerUserId, sourceType(messenger|task|calendar|listing), sourceId, chatId?, messageId?, snippet?, readAt?, createdAt. @@unique[messageId,mentionedUserId] (re-edit no dup). idx[mentionedUserId,createdAt] + [sourceType,sourceId]. FKs to User onDelete Cascade.**
- Migrations: messenger_core, messenger_dm_key, messenger_phase2, drop_task_comments, chat_parent_unique, **20260601140000_mentions**. TaskComment DELETED.
- Presence: NO schema (Redis only). User.onlineStatusMode (everyone|contacts|nobody) + cardVisibility.onlineStatus flag govern privacy.

## Phase 5 Mentions Hub (details)
- Mentions are PICKED (users have no @handle), stored inline as token `@[Имя](userId)`. Parser+token live in @superapp/shared: `parseMentions(content)` (regex `/@\[([^\]]{1,80})\]\(([0-9a-fA-F-]{36})\)/g`, dedup+cap MENTION_LIMITS.maxPerMessage=20), `mentionToken(name,userId)`. MENTION_LIMITS{feedPageSize:30, snippetLength:140}. Types: MentionSourceType, MentionItem, MentionFeed, MentionCandidate. notification type 'mention.received' (icon '@', pushByDefault).
- **MentionsService** (best-effort, NEVER throws — wrapped try/catch so it can't break sendMessage/editMessage):
  - recordMessageMentions: parse → drop author → **security-filter to CURRENT active chatMember (leftAt:null)** (token is attacker-controlled, membership table is not — non-members CANNOT be mentioned/notified) → findMany existing for (messageId, in keptIds) → createMany skipDuplicates ONLY the diff → notify ONLY newly-created (edit adds new mention notifies only the newcomer, never re-notifies). Hooked in messenger sendMessage (after commit+emit) AND editMessage (new content).
  - listFeed(userId,cursor): keyset (createdAt desc,id desc), batch-resolve mentioner names + chat titles (no N+1), DM has no title → fallback mentioner name. urlFor: messenger→/messenger?chat=, task→/tasks/:id, calendar→/calendar?event=, listing→/shop?listing=.
  - markRead(userId, ids?): updateMany scoped to mentionedUserId (can't mark someone else's); empty ids = all unread.
  - mentionableMembers(viewer,chatId,q): access.can('chat.view') → other active members minus viewer, filter by name substring, cap 20.
- SOURCE-AGNOSTIC: feed/markRead/candidates serve all sourceTypes; Ф5 only PRODUCES messenger mentions (task/calendar later, no hub change).
- Web: **MentionInput** (composer textarea + @-autocomplete; trigger /(^|\s)@([^\s@]{0,30})$/ before caret; debounce 150ms getMentionable; popover above; arrows/Enter/Tab select, Esc close; onMouseDown select so blur doesn't beat it; inserts mentionToken+' ' + restores caret; preserves Enter-send/Shift+Enter/typing-indicator). **mention-render** renderMessageContent(content,currentUserId)→text+MentionChip nodes (mine=primary, others=secondary), stripMentions→`@Имя` for previews (used in ChatList preview + hub snippet). **/mentions** hub page (feed, unreadCount+plural, "Прочитать все", optimistic per-row mark-read + deep-link, cursor "Показать ещё", empty state). **MentionsNavLink** wax-red badge in messenger nav, fed by **useMentionsUnread** (shared cache key mentionsFeedKey ['mentions','feed'], 60s poll + focus). Dashboard "Упоминания" tile (static, no badge).
- **Deep-link to exact message**: hub openMention builds `/messenger?chat=<id>&msg=<messageId>` for messenger mentions (other sources use server url). messenger/page reads ?msg → highlightMsgId → Conversation prop highlightMessageId + onHighlightConsumed. Conversation effect: once msg in loaded window, getElementById(`msg-<id>`) scrollIntoView center + flash ~2s (secondary wash), consume. If older than loaded page → just opens chat (MVP). MessageBubble has id=`msg-<id>` + highlighted prop.
- **Badge decrement fix**: hub markOneReadLocally always `unreadCount-1` (caller only invokes for unread row) — works for paged-in rows too (count is global on first page).

## GOTCHAS
- Rebuild @superapp/shared before api build when shared types change.
- prisma generate EPERM (Windows) when node holds query_engine DLL → kill node(dist/main,nest), delete .prisma/client/*.tmp. `pnpm --filter exec prisma` flakes → run `node apps/api/node_modules/prisma/build/index.js generate`.
- prisma migrate dev non-interactive fails → hand-write migration.sql + migrate deploy.
- Gateway/services must NOT import @nestjs/config → process.env.
- Never full `tsc --noEmit` on API (OOM) — use nest build. Web tsc fine.
- Parallel PowerShell batch: a non-zero exit cancels sibling tool calls → run build/verify/start ONE per message.
- raw `node dist/main` + verify-*.cjs need apps/api/.env loaded into the PS process (DATABASE_URL, JWT_SECRET). socket.io-client for verify scripts: require from apps/web/node_modules.
- login response has NO user obj → id from GET /users/me.
- preview controlled inputs: native-setter + dispatch input event; login submit via form.requestSubmit() (button .click races React state). Preview session token can drop on HMR → re-login.
- Browser smoke this session BLOCKED: Claude-in-Chrome extension not connected (3 retries); preview can't take port 3000 (held by user's dev server, needed for CORS/OAuth). Use Chrome extension when user has it open.
- ASYNC PROJECTION RACE: never gate a sync read on tuples written only by an async listener → project at the mutation synchronously.
- e2e contact setup: deterministic Prisma contactLink upsert, NOT the invite flow.
- presence:<uid>:ctx 60s cache → presence e2e must use unique event titles + del the ctx Redis key between assertions + set calendar share BEFORE reading. Calendar share body = {sharedWithUserId, accessLevel}.
- mention token regex assumes 36-char UUID — user ids ARE UUIDs (verified). If id format ever changes, update BOTH shared parseMentions AND web mention-render TOKEN.
