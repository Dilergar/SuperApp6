BUILT 2026-07-17. verify-calls 8/8 + verify-office 51/51 (после доработки «История») + регрессии (messenger 29, group 31, richcards 29, staff) зелёные + браузер с живым LiveKit (реальный connect, вебхуки в журнал, чат live, «Завершить для всех» → ROOM_DELETED-экран; секция «История» + чат завершённой встречи).

## Решения (грилл)
- **LiveKit OSS self-host** (`livekit/livekit-server:v1.13`, Apache-2.0) — пользователь сперва требовал «свой SFU 1в1 Google Meet, юристы решат с Google»; разубеждён фактами: код Meet закрыт и не продаётся, но ВСЯ технология Meet открыта самим Google (WebRTC/GCC/simulcast/SVC/Opus RED — роялти-фри стандарты, платить не за что); свой SFU = 1,5–3 года команды; LiveKit = те же технологии готовыми (~12 из 15 техник устойчивости из коробки/флагом; на нём ChatGPT Voice). Прецедент-паттерн: whisper для voice, SeaweedFS для files.
- Чат встречи = контекстный чат мессенджера (в v1); запись — Ф3 (egress тяжёлый); доступ — только команда ws (Стажёр+, Подрядчик изолирован); гости извне — позже.

## Архитектура движка (core/calls)
- **CallSession** (`roomName='call_<id>'` @unique; refType+refId полиморфно БЕЗ FK — паттерн FileLink) + **CallSessionParticipant** (журнал; «открытая строка» = leftAt IS NULL). **Partial unique «одна active на ref» дописан РУКАМИ в миграцию** `20260716204209_calls_office_engine` (Prisma не умеет WHERE-индексы; зеркало-комментарий в schema.prisma) — гонка первых токенов гасится P2002→перечитать победителя.
- **CallsRefRegistry** (копия FilesRefRegistry): `register(refType, {canJoin, canModerate, onJoinAuthorized?, resolveWorkspaceId?})` в onModuleInit потребителя. `onJoinAuthorized` = СИНХРОННАЯ материализация участника при выдаче токена (шина at-most-once не годится; токен — единственный вход, покрытие 100%).
- Токен подписывается ЛОКАЛЬНО (AccessToken; identity=userId, name, roomAdmin=canModerate, TTL 600с) — verify-скрипты живут без сервера. `livekit-server-sdk@^2.15` грузится обычным require из CJS-Nest (проверено).
- **Вебхук** `POST /api/calls/livekit/webhook` @Public+@SkipThrottle: WebhookReceiver проверяет подпись по СЫРОМУ телу → **точечный `app.use('/api/calls/livekit/webhook', express.raw({type: () => true}))` в main.ts ПОСЛЕ alias-мидлвари** /api/v1→/api (оба префикса покрыты; глобальный json Nest вешает в listen() — позже). `express` добавлен ПРЯМОЙ зависимостью apps/api (pnpm строгий — прецедент multer). room_finished закрывает СЕССИЮ (не сущность); идемпотентность: status-guarded updateMany (count 0 → выход), open-row-check для joined.
- Ошибки LiveKit двухуровнево: deleteRoom/listRooms — best-effort (БД истина; reconcile при недоступности ПРОПУСКАЕТ прогон, не закрывает вслепую); kick/mute — БРОСАЮТ 502 («модерация не врёт»).
- CallsCron */5 мин `redis.withLock('cron:calls-reconcile')`: активные старше 10 мин без комнаты в listRooms → ended (ловит потерянный room_finished и «токен выдан — никто не пришёл»).

## Устойчивость к слабой сети (2026-07-18, ресёрч + внедрение)
Ресёрч (3 агента): Meet = сумма открытых стандартов (GCC, simulcast/SVC, Opus RED 2-3x избыточность, NetEQ, приоритет аудио, TURN/443 — всё IETF/W3C, патентно безопасно); Bitrix24 (Voximplant SFU) и Телемост (свой SFU) — тоже WebRTC, отставание = тюнинг. «Ускоренный звук после лага» Telegram = **NetEQ Accelerate** (WSOLA time-stretch без изменения тона) — встроен в браузерный WebRTC, у веб-клиента LiveKit работает автоматически (webrtc-internals: `removedSamplesForAcceleration`). Opus 1.5 DRED/deep PLC и Lyra — НЕ в браузерах (только нативные клиенты), неприменимо. Krisp = только LiveKit Cloud; self-host альтернатива = RNNoise-WASM.

**Внедрено:**
