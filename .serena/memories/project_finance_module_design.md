# «Финансы» (B2C) — ПОСТРОЕН 2026-07-02 (Ф1–Ф8, все фазы; дизайн заперт грилл-сессией)

Личный + семейный управленческий учёт. **Редактируемая учётная книга со СТРУКТУРОЙ двойной записи** (Firefly III / QuickBooks: «всё — счёт», операция = пара from→to) — записи о ВНЕШНИХ деньгах. Кошелёк-леджер коинов НЕ тронут (immutable расчётный слой); Финансы показывают его read-only проекцией. Правки разрешены: soft-delete + FinAuditLog (кто что менял).

## Архитектура (как построено)
- `apps/api/src/modules/finances/`: finances.service.ts (вся логика), finances.controller.ts (тонкий, Zod, AI-ready), finances-cards.provider.ts (rich cards + quick action), finances.cron.ts (*/30 мин Redis-лок), finances.events.ts (contact.removed → отзыв прямых finbook-грантов), finances.module.ts (+строковый токен 'FinancesService' для нод Процессов).
- Prisma (миграции `finance_core`, `finance_debt_reminder`): FinBook (@@unique ownerType+ownerId; user|workspace) · FinAccount (kind asset|liability|expense|income|equity; parentId-дерево категорий 2 ур.; liability несёт debtTotal/Monthly/Months/DueDay/ClosedAt/RemindedAt) · FinTransaction (from→to, amount BigInt тиын, amountTo для обмена, occurredOn @db.Date, personUserId+personName, createdById, source manual|recurring|process|service|import, deletedAt) · FinBudget (@@unique book+category+period) · FinPerson («Близкие») · FinRecurringRule (interval monthly|weekly, autoRecord, nextRunAt) · FinAuditLog (FK-free).
- Балансы = SUM raw-SQL (inflow COALESCE(amount_to,amount) − outflow) — материализация не нужна на личных объёмах.
- resolveBook(userId, bookId?, need) — единственный chokepoint прав: owner кодом; чужие — core/access `finbook` (editor⇒viewer; capabilities finbook.view/edit; EPOCH_FANOUT + circle-fanout). Доступом управляет только владелец; одна роль на принципала (grant поверх revoke обеих).
- Пары счетов: asset→expense (расход) · income→asset (доход) · asset→asset (перевод; кросс-валюта = amountTo, повторы кросс-валюту запрещают) · liability→expense (рассрочка-покупка ПОЛНОЙ суммой в месяц покупки — решение юзера, бьёт лимит) · liability→asset (кредит; received<total → авто-расход «Проценты по кредитам») · asset→liability (платёж = погашение, НЕ расход — секция «Платежи по долгам») · equity↔asset (только системно: openingBalance/set-balance).
- План-факт: пороги 80/100% проверяются СИНХРОННО после коммита (fire-and-forget гонялся с быстрыми тратами → двойной exceeded) — уведомление только при пересечении именно этой операцией.
- Долги: «Оплачено» в 1 тап (дефолт monthly, кэп остатком, remaining=0 → debtClosedAt+нотиф); напоминание в день платежа (клиппинг к концу месяца, дедуп debtRemindedAt раз в день).
- Повторы: крон клеймит срабатывание атомарным сдвигом nextRunAt (updateMany WHERE nextRunAt=старый) → двойная запись невозможна; autoRecord=false → notify finance.recurring.due + кнопка «Записать сейчас» (POST /recurring/:id/record-now).
- Rich cards: `fin_transaction` + `fin_month` (refId=`bookId:YYYY-MM`) — модель-СНИМОК (движок rich-cards и так кладёт payload шарящего в сообщение; живой рендер = null без finbook.view → чужой видит снимок, но не «Открыть»). Quick action `finance.add-expense` (AddExpenseModal в QuickActionModals.tsx + KNOWN_KEYS).
- Коин-лента `GET /finance/coins`: LedgerTransfer (posted|post_pending) моих user-счетов → обогащение: EscrowAgreement.refType → task (title задачи, href /tasks/id) | order (titleSnapshot, краудфандинг), counterparty (user→PersonChip, workspace→«Казна: X», issuance→mint/burn). НЕ шерится (кошелёк личный).
- Календарь: слой `finance` (CalendarLayer + CalendarFinanceItem c полем `start`!) — CalendarModule imports FinancesModule (ребро в карте CLAUDE.md), getPaymentsForCalendar раскрывает дни платежей (долги dueDay с клиппингом + повторы) в диапазоне; веб: тумблер «₸ Платежи», isFinance-гарды в calendar-lib (all-day, не draggable, клик → /finance).
- Процессы: нода `finance.record` (process-action-nodes.ts, ACTION_PROCESS_NODES) — recordOperationForBook(workspaceId,…): книга организации лениво, категория ПО ИМЕНИ лениво, счёт «Касса» лениво, сумма в тенге×100, source='process', от имени инициатора. Событие `finance.transaction.created` на шине (payload несёт workspaceId для org-книг) + тип в PROCESS_EVENT_TYPES → триггер «при расходе > X — согласование».
- Веб `/finance` (5 файлов: page.tsx, finance-report.tsx, finance-debts.tsx, finance-access.tsx, finance-coins.tsx, finance-lib.ts): левая колонка Счета(+set-balance)/Долги/Повторы/Близкие/Категории; правая — вкладки Лента (быстрый ввод: Расход|Доход|Перевод + «на кого» чипами Близких/EntitySelector; лента по дням, правка/удаление/💬-в-чат) | Отчёт (план-факт бары, доходы, по людям, тренд 6 мес, 💬 «итоги месяца») | Коины. Переключатель книг (shared-with-me, PersonChip владельца) + «Доступ» (AccessModal: user+circle, роли, ×). Режим viewer = только просмотр (все формы/кнопки скрыты). Автор чужой записи — PersonChip «внёс(ла)». RQ-ключи в lib/queries.ts (finance*).
- Уведомления (категория finance): budget.warning/exceeded, debt.payment_due, recurring.due/recorded, book.shared.

## Решения грилла (зафиксированы)
Одна книга целиком · лимиты по категориям (не YNAB) · валюта на счёт без курсов (обмен = 2 суммы) · дерево категорий, подкатегория опциональна · рассрочка = расход полной суммой сразу (категория+«на кого» в момент покупки) · платёж по долгу = напоминание+1 тап (НЕ авто) · повторы с переключателем авто/напоминание · editor правит всё (Дзен-мани) · «на кого» видно всем с доступом · шеринг людям И Группам · коины только владельцу · «на кого» у расходов И доходов · кредит деньгами поддержан · «на кого» = НЕ тег, а personUserId+PersonChip.
