# Сотрудники (Staff, B2B) — построен 2026-06-11

Полноценный сервис управления сотрудниками организации. Дизайн утверждён интерактивным гриллом (все развилки — решения пользователя).

## Модель (несущие решения)

- **Одна роль прав на организацию** (UserRole, единый источник) + **статус обучения на каждом назначении должности** — НЕ роль-на-должность. Лестница `WORKSPACE_ROLE_RANK` (shared/constants/roles.ts): `contractor(1) < trainee(2) < staff(3) < manager(4) < admin(5) < owner(6)`.
- **Найм ВСЕГДА в Стажёра** (`WORKSPACE_HIRE_ROLE='trainee'`): приглашение НЕ несёт выбора роли (колонка `role` осталась, форсится trainee при accept). Повышение вручную; позже — бизнес-процессы/Додзё.
- **Стажёр сейчас = метка** (решение пользователя): функционально равен Сотруднику, реальные гейты придут с Додзё. Но manager+-гейты на него действуют (не управляет справочниками).
- **Подрядчик (`contractor`, экс-`guest`)** — Коллаб-модель Bitrix24: НЕ приглашается и НЕ назначается вручную (нет в ASSIGNABLE; updateMember → 400) — только программно будущими сервисами (Тайный гость, UGC-блогеры). Изолирован: `assertReachable` в ws-контексте требует командных ролей (trainee+) у ОБЕИХ сторон; ростер/справочники → 403; должности не назначаются; в `workspace#member` НЕ проецируется. Сейчас прав ноль; реальные точечные доступы (его задача/чат/событие) выдаст владеющий сервис.
- **Членство в отделе — производное от должности** (`StaffPosition.departmentId`), модель штатного расписания 1С. Прямого назначения «человек→отдел» НЕТ.
- **Отделы — дерево в данных** (`parentId`), UI пока плоский с выбором родителя; цикл при перемещении → 400.
- **Назначение** = `StaffAssignment(workspaceId, userId, positionId, branchId?, status, assignedBy)`; несколько на человека; та же должность в двух филиалах = два назначения; статус `training|certified` (фундамент Додзё; manager+ аттестует вручную, событие `workspace.position.certified`). Partial unique (ws,user,position) WHERE branch IS NULL + композитный unique с branch.
- **Лестница управления**: справочники/назначения/аттестация/наём — **manager+** (iiko: управляющий нанимает сам); роли и увольнение — admin+; **админа назначает/снимает/увольняет ТОЛЬКО владелец, админ не трогает админов**.
- **Без лимитов найма**: убраны maxInvitationsPer24h и resendCooldownHours из WORKSPACE_LIMITS (решение «нанять всех за день»); потолок pending 500 — анти-мусор. Bulk-найм пользователь отклонил (по одному).
- **Руководители отделов/филиалов — ПОЗЖЕ** (вместе с канвасом оргструктуры на нодовом движке n8n-модели).

## Реализация

- API: `modules/staff/` (StaffService/Controller/Module) — `/api/workspaces/:id/staff/*`: GET справочники (+счётчики), CRUD departments/positions/branches (дубль имени → 409; удаление должности/филиала с назначениями → 409; удаление отдела отцепляет должности и поднимает подотделы в корень), POST members/:userId/assignments, PATCH/DELETE assignments. Тонкие контроллеры + Zod (`validation/staff.ts`) — AI-ready.
- WorkspacesService: invite `{phone, positionId?, branchId?, message?}` (manager+, валидация справочников), accept → членство + trainee + `StaffService.createAssignmentTx` в ОДНОЙ транзакции (проекция после коммита), listMembers + `assignments[]` (`getAssignmentsByUser`), updateMember `{role}` (правила лестницы), removeMember/leave → каскад `removeAllAssignmentsForUser`.
- core/access: проекция `position#holder@user` / `branch#member@user` / `department#member@user` **с closure предков** (`applyStaffDiff` per-workspace + глобальный `reconcileStaff` в AccessReconcileCron); маппинг ролей `WS_ROLE_RELATION` staff|trainee→member, contractor не проецируется; `EPOCH_FANOUT` department/position/branch → [self, card, showcase, calendar]; `staffEntityDeleted` revokeResource.
- contacts.assertReachable (ws-ветка): роли вместо WorkspaceMember-строк, фильтр `TEAM_WORKSPACE_ROLES` — проверяет И инициатора (подрядчик не инициирует), И цели.
- Prisma: `staff_departments/staff_positions/staff_branches/staff_assignments`; WorkspaceMember БЕЗ position/department (текст мигрирован в сущности+назначения certified); WorkspaceInvitation: positionId/branchId FK SetNull, role default trainee; data-миграция guest→contractor. Миграция `20260611100000_staff_directories`.
- Web `/workspaces/[id]/members` (URL прежний): вкладки Сотрудники (фильтры; ростер — **`StaffPersonCard`** в PersonCard.tsx: ТА ЖЕ карта что в Окружении — L-грид auto-fill minmax(200px,1fr) gap-6, **клик → XL-оверлей** (ExpandedCard переиспользован, onWrite стал optional); **бейдж карты = Должности (`positions[]`, join «/»), филиалы — ОТДЕЛЬНЫЕ чипы 📍 (`branches[]`) под бейджем** (по просьбе пользователя: должность и филиал не сливать); под телом ДВЕ явные кнопки — «Написать» (контур secondary, POST /messenger/chats/dm с X-Workspace-Id → push `/messenger?chat=`) + «Управлять» (заливка primary, manager+) → модалка роль/назначения/увольнение; секция Подрядчики admin+; fallback `cardOf()` от устаревшего RQ-кэша без `card`) | Должности | Отделы | Филиалы | Приглашения (форма найма — клон b2c «Добавить в окружение» 1в1: номер → debounce-lookup `/users/lookup` («Диана Н.» + PersonAvatar) → ДВА блока чипами в стиле RolePicker (`ChipPickerBlock`): «Должность» single + «Филиалы» MULTI → «всегда Стажёр». `WorkspaceInvitation.branchId`→`branchIds String[]` (миграция `invitation_multi_branch`, FK снят — scalar-массив, имена резолвятся батчем в `serializeInvitations`); accept создаёт назначение на КАЖДЫЙ филиал; `InviteMemberRequest.branchIds?[]`, DTO `branchIds/branchNames`). React Query на ключах `workspace*Key` (lib/queries.ts). **Решение пользователя 2026-06-12: кнопку «Аттестовать» и бейдж статуса УБРАЛИ из UI** (пустые до Додзё) — API `PATCH assignments {status}` остаётся, Додзё подключится к нему.
- **«Видимость в Компаниях» (2026-06-12)**: `users.companyCardVisibility` (Json, миграция `user_company_card_visibility`, та же форма CardVisibility) — настраивается в `/profile/form` ВТОРЫМ блоком тумблеров рядом с «Видимостью по умолчанию»; `GET/PATCH /users/me` несут поле; `listMembers` отдаёт `member.card` (тип `ContactUserCard`) — поля профиля, маскированные флагами ВЛАДЕЛЬЦА (скрытое → null; имя/фамилия/телефон всегда; age считается на бэке). Карта ростера наполняется этими данными — b2c-механика видимости перенесена на компанию 1в1. Проверено в verify-staff (52 чека: + PATCH/GET companyCardVisibility, маскирование bio/city в ростере).
- **EntitySelector**: `department`/`position`/`branch` зарегистрированы в `lib/entities.ts` — workspace-скоупные лоадеры (`loadEntities(type, {workspaceId})`, кэш `type|wsId`, один HTTP `/staff` на три типа), `EntitySelector` принимает `context={workspaceId}`. ROLE_LABELS на вебе — из shared `WORKSPACE_ROLES` (4 локальные копии удалены).
- Уведомления: `workspace.position.assigned`, `workspace.position.certified` (+listener cases); `workspace.invitation.received` без роли («приглашает вас на работу»); `workspace.role.changed` шлёт русский лейбл.
